## 1. PROJECT OVERVIEW
The application is a web-based **Cyber Security Operations Center (SOC) Console** that monitors TLS Client Hello handshakes, decodes TLS handshakes into the state-of-the-art **JA4 Fingerprint format** (Segment A, Segment B, Segment C), matches them against a local database of known signatures, triggers real-time Telegram Bot alerts, and generates custom Python network-sniffing scripts for standalone deployment on target machines.

### Core Key Features:
1. **Interactive SOC Dashboard**: A dark-themed, sleek user interface featuring a live pipeline of network ingress, dynamic status indicators, and beautiful negative-space typography.
2. **Local Signature Management Database**: A fully managed local database where administrators can add new JA4 fingerprints, tag them with custom risk levels (Safe, Warning, Critical), and specify associated tools/malware names (e.g., Cobalt Strike, Tor Browser, Metasploit, standard browser).
3. **Deep Packet Handshake Inspector**: Breaks down any highlighted packet into its constituent JA4 components:
   - **Segment A**: Transport metadata (protocol type, TLS version, destination port, ciphers/extensions counts, ALPN indicator).
   - **Segment B**: Hexadecimal representation of sorted active cipher suites, hashed and truncated to 12 characters.
   - **Segment C**: Hexadecimal representation of sorted active extensions, hashed and truncated to 12 characters.
4. **Telegram Alerting Engine**: Ties the detection pipeline directly to Telegram channels. Real-time threat alerts are posted dynamically to the configured Telegram bot.
5. **Interactive JA4 Signature Sandbox**: An educational playground where users can input custom TLS protocols, ALPN strings, hex ciphers, and hex extensions to dynamically compute a JA4 signature on-screen and check it against database definitions.
6. **Autoconfigured Python Exporter**: Dynamically bundles and configures a standalone python script (`ja4_ids.py`) pre-injected with the active Telegram credentials that can be downloaded to run live sniffing on production servers using `Scapy`.

## 3. SIGNATURE MATCHING LOGIC (NO GEMINI API)
When a raw packet is processed (either through live local sniffing or UI simulation):
1. Parse the TLS Handshake payload to extract JA4-related parameters.
2. Formulate the formatted JA4 string: `[Segment A]_[Segment B]_[Segment C]`.
3. Query the local SQLite database for a match on the `ja4` string.
4. **Matched Result**:
   - If a record is found, associate the packet with the registered `threat_name`, `risk_level`, and `description`.
   - If the threat matches `Warning` or `Critical` status, trigger the Telegram Bot alerting routine dynamically.
5. **Unmatched Result**:
   - If no database match is found, classify the packet with `Safe` status, marking the app match as "Unknown client" or "Unregistered Host Fingerprint".
   - The UI should provide a button next to unrecognized captures in the pipeline: **"Add to Database Signatures"**, pre-filling the registration form with the capture's JA4 string, and allowing instant classification.

def compute_ja4_segment_a(proto, tls_ver, port, num_ciphers, num_exts, alpn):
    """
    Format Segment A: 
    1. Protocol (t for TCP, q for QUIC) -> char 1
    2. TLS Version (e.g. 13 for 1.3) -> chars 2-3
    3. Destination Port (d if 443, o if other) -> char 4
    4. Num Ciphers (2 chars, padded with 0, max 99) -> chars 5-6
    5. Num Extensions (2 chars, padded with 0, max 99) -> chars 7-8
    6. ALPN (2 chars, dynamic index: e.g. h2 -> h2, none -> 00) -> chars 9-10
    """
    p = 't' if proto == 'tcp' else 'q'
    v = tls_ver if len(str(tls_ver)) == 2 else '13'
    dp = 'd' if port == 443 else 'o'
    nc = f"{min(num_ciphers, 99):02d}"
    ne = f"{min(num_exts, 99):02d}"
    
    # Simple ALPN representation
    alpn_val = '00'
    if alpn:
        if 'h2' in alpn: alpn_val = 'h2'
        elif 'http/1.1' in alpn: alpn_val = 'h1'
        else: alpn_val = alpn[:2]
        
    return f"{p}{v}{dp}{nc}{ne}{alpn_val}"

def compute_truncated_sha256(values_list):

    # Sort values, filter grease codes if any, and hash
    sorted_str = ",".join(sorted(values_list))
    hasher = hashlib.sha256(sorted_str.encode('utf-8'))
    return hasher.hexdigest()[:12]
